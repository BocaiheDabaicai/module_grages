# 关于

## 项目介绍

本项目创建的初衷是为了提高开发效率，减轻开发人员的负担，在已有的标准基础前端结构下进行项目开发、项目维护与更新。

## 项目配置

本项目使用`vitePress`进行实现，在2.0.0版本下打造而来，项目完整配置需求如下：

- `vue`：所需版本`3.5.26`及以上

- `shiki`：所需版本`3.20.0`及以上

- `sass`：所需版本`1.97.1`及以上

- `element-plus`：所需版本`2.13.0`及以上

- `vite`：所需版本`7.0.0`及以上

## 项目初始化

1. 项目构建

获取项目的资源依赖包

```shell
pnpm i
```

2. 项目启动

启动项目，已查看当前开发内容

```shell
npm run dev
```

3. 项目打包

打包项目资源，以便于放置在服务器对拥有`VPN`人员共享

```shell
npm run build
```

4. 项目预览

预览打包好的项目内容和界面

```shell
npm run preview
```

## 项目愿景

打造一款可以快速上手的前端库，能够利用里面的模板或者是组件，构建出自己需要的模板或者组件，构建方式不局限于模板，而是更加开放包容的设计理念，即**可以多个组件多个模板**共同组建出自己需要的界面实现效果。

为能够达到想要实现的效果，总共会实现以下理念的构想：

1. 后台系统的前端模板库：实现条件筛选、信息输入、数据采集、信息监控四种主要核心模块，后期不定更新继续拓展前端库内容。

2. H5界面的前端模板库：实现动态生成界面、登录与账号管理、主页、条件筛选H5、信息输入H5，后期不定更新继续拓展前端H5库内容。

3. 前端功能库：实现接口请求、本地化存储、二维码、图片上传下载、扫码、支付、语音输入信息、地图、定位、打印模板、可视化图表功能，后期不定更新继续拓展功能库内容。

4. AI生态库：实现前端库学习、内部系统架构学习、指令生成界面、指令生成组件、指令生成系统、自动化界面弹窗提示、信息预警提示、自动化生成可视化面板、自我迭代系统、自我实现需求，目前处于构想阶段，后期逐步尝试并实现。

5. 设计构建界面：实现通过可视化拖拽和`JSON`参数设计，一键得到需要的界面或组件，并生成对应的 Vue SFC 代码。低代码平台核心功能已上线，后期继续完善组件库覆盖和交互体验。

6. 开发业务统计：实现自动计算业务更新的内容量、代码更新的内容量、后台系统的开发数量、模块开发的数量、已实现功能的可视化表达、日开发量统计、周开发量统计、月开发量统计、文档报表的统计，最后是代码更新、文档更新、项目更新、模块更新、功能更新的关联性统计与业务量表达，目前处于构想阶段，后期逐步尝试并实现。
